# TerminalIN 引き継ぎ — 2026-04-03 01:30

## 1. プロジェクト概要

**TerminalIN** — macOS向けターミナルウィンドウマネージャー。Electronアプリで、埋め込みターミナル(xterm.js+node-pty)と外部Terminal.appウィンドウの両方を1つのワークスペースに統合する。

**ビジョン**: ブラウザのタブのように、様々なターミナルウィンドウを1つのコントロールパネルでグループ管理する。

## 2. 技術スタック

- **Electron 41** (macOS, `titleBarStyle: 'hiddenInset'`, `vibrancy: 'sidebar'`)
- **node-pty 1.1** — 埋め込みシェルセッション
- **@xterm/xterm 6.0 + @xterm/addon-fit 0.11** — ターミナルレンダリング
- **Swift バイナリ4本** (Accessibility API + CGWindowList):
  - `list-windows` — Terminal.appウィンドウ一覧取得 (CGWindowListCopyWindowInfo)
  - `batch-move` — 複数ウィンドウ一括移動 (AXUIElement, `_AXUIElementGetWindow` private API)
  - `raise-windows` — ウィンドウをAXRaiseで前面に
  - `move-window` — 単一ウィンドウ移動 (旧版、現在未使用)
- **nodeIntegration: true, contextIsolation: false** — rendererから直接require()

## 3. 現在のアーキテクチャ

### ファイル構成 (アクティブなもの)
```
main.js          — Electron main process (pty管理 + 外部ウィンドウIPC)
preload.js       — IPC bridge (現在はcontextIsolation:falseなので実質不使用)
workspace.html   — メインUI (コントロールパネル)
app.html         — 旧バージョン、未使用
list-windows.swift / list-windows    — Swift source / compiled binary
batch-move.swift / batch-move        — Swift source / compiled binary
raise-windows.swift / raise-windows  — Swift source / compiled binary
package.json
```

### 現在の動作
1. **Electronウィンドウ** = 左サイドバー型コントロールパネル (320×650, alwaysOnTop)
2. **埋め込みターミナル**: xterm.jsカードが展開/折りたたみ可能
3. **外部ターミナル**: 800msポーリングでTerminal.appウィンドウを検出。「Snap」でElectronウィンドウの右横に2×2タイル配置。「Release」で元位置に復帰
4. **ウィンドウ追従**: Electronウィンドウのmoveイベントで16ms throttledで外部ターミナルをretile

### 外部ターミナル識別
- `CGWindowID` (`windowNumber`) を安定ID として使用
- batch-move/raise-windowsは `_AXUIElementGetWindow` private APIでCGWindowIDマッチング
- フォールバック: title → title prefix → windowIndex

### 日本語環境の注意
- Terminal.appのプロセス名は **「ターミナル」** (CGWindowListのownerName)
- list-windows.swiftの `terminalApps` Setに含まれている

## 4. 既知の課題

### 解決済み
- ~~windowIndex不安定~~ → CGWindowIDに切替済み
- ~~backdrop z-order問題~~ → サイドバー方式に変更
- ~~AppleScript遅すぎ(30+ウィンドウ)~~ → Swift binary化
- ~~setIgnoreMouseEvents不安定~~ → 透明領域クリック透過に変更

### 残存
1. **外部ターミナルの遅延**: Electronウィンドウ移動時、Swift batch-move呼び出しに数十ms遅延。根本的にはmacOSのウィンドウ間連携の限界
2. **外部ターミナルの入力**: Electronウィンドウ(alwaysOnTop)が外部ターミナルの上に被る場合がある。現在は横並びで回避
3. **不要ファイル多数**: index.html, backdrop.html, control.html, frame.html, move-window.swift, list-windows-debug.swift は開発過程の産物。クリーンアップ未実施

## 5. 次のアクション（ユーザーの要望）

**「このようなworkspaceウィンドウを複数作れるようにしたい」**

### 実装方針案
1. `main.js` でBrowserWindowを複数管理する Map/Array 構造に変更
2. 各ワークスペースが独立したsnappedExternals Mapを持つ
3. メニューまたはキーボードショートカットで新規ワークスペース作成 (Cmd+N)
4. 各ワークスペースウィンドウが独立してmove追従
5. 外部ターミナルは1つのワークスペースにのみスナップ可能（排他制御）
6. ワークスペース間でターミナルをドラッグ移動（将来機能）

### 具体的な変更箇所
- `main.js`: `let win` → `const windows = new Map()` (id → {win, snappedExternals, ptys})
- `createWindow()` を `createWorkspace(id)` に変更
- IPC handlerをwindow IDで振り分け
- workspace.htmlは変更不要（各ウィンドウが独立してロード）
