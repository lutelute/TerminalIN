## 2. 今回のセッションで完了したこと

### パフォーマンス最適化
- `atomicWriteJSON` 非同期化 (event loop ブロック解消)
- `findWorkspace` → WeakMap O(1)、`isExternalSnapped` → グローバル Map O(1)
- poll handler: identity チェックで変化なし時 IPC スキップ
- renderer: xterm コンテナ保持で再マウント回避
- renderer: snappedExternals のタイトルを live window から同期更新 (タイトル表示バグ修正)

### daemon TCC 権限問題の解決 (最重要)
- **daemon を `Contents/Resources/` → `Contents/MacOS/` に移動**
- macOS TCC が `MacOS/` 内のバイナリを親アプリの一部と認識 → Accessibility 権限を自動継承
- パッケージ版の snap: 300-3000ms → **8-56ms** (dev 版と同等)
- ユーザーによる daemon 個別 Accessibility 追加が不要に

### sidebar ドラッグのリアルタイム追従
- `win.on('move')` で 16ms throttle、daemon に fire-and-forget で move 送信
- ドラッグ中は poll をスキップ (daemon 競合防止)
- `daemonMoveFireAndForget()` 関数追加 — 応答を待たずに stdin に書き込む

### UI 改善
- vibrancy: `'titlebar'` (最も明るい) + `visualEffectState: 'active'`
- 非アクティブ時: CSS `::after` で灰色マスク (opacity 0.18) をフェードイン
- テキスト色を全体的に濃く (アクティブ時にくっきり読める)

### ビルドインフラ
- `scripts/build-daemon.sh`: daemon.swift 未変更時はコンパイルスキップ (CDHash 安定化)
- `scripts/install.sh`: daemon inode 保持インストール (TCC 権限維持)
- `package.json`: predist/prebuild に build-daemon.sh 追加
- daemon.swift: 未使用 `readCompositorPosition` 削除

## 6. コンテキスト・メモ

### ユーザーの好み
- **動作の一体感が最重要** — snap したターミナルが workspace に「貼り付いて固定されている」感覚
- 細かい最適化より体感の滑らかさを重視
- 「基本 Electron (dev) で開発」— dev モードで確認してからパッケージ版をビルド
- AtelierX は TiN と競合していない (気にしなくていい)

### 開発フロー
1. `npx electron . --dev` で開発・確認
2. `npm run dist && bash scripts/install.sh` でパッケージ版テスト
3. electron-mcp-server で screenshot/CDP 操作可能 (port 9222)
4. main process ログは dev モードの stdout か、直接バイナリ実行で取得

### 試行錯誤の教訓
- raise-all や retile を削ると「一体感」が壊れる → 削ってはいけない
- daemon AX のラッチ (一度 untrusted → 永久 fallback) は危険 → 毎回試行が安全
- osascript のタイムアウト 3s は長すぎ → 1.5s に短縮済み
- `Contents/MacOS/` 配置が TCC の鍵 — `extraResources` ではなく `extraFiles` を使う
