## 6. 既知の問題（最重要）

### 6.1 パッケージ版でのSnap/移動が効かない or 位置がずれる
- **原因推定**: 座標系の不一致。ユーザーが「ディスプレイ認識がうまくできていないかも」「一致していないかも」と発言
- daemon の `handleMove` はAXUIElement APIを使用。パッケージ版では権限不足で `moved: 0`
- osascript フォールバック `set bounds of window id N to {left, top, right, bottom}` 追加済み
- **しかし位置がずれる**: Electron `getBounds()` の座標と AppleScript `bounds` の座標系が異なるディスプレイ構成で不一致の可能性

### 6.2 座標系の不一致（未解決・最重要調査項目）
- CGWindowList 座標: (0,0) = メインディスプレイ左上、y軸下方向
- AppleScript `bounds`: {left, top, right, bottom} — 同じく左上起点
- Electron `getBounds()`: macOS スクリーン座標 — **Retinaスケーリングの影響あり?**
- **ユーザーはM4 Max MacBook Pro + 外部ディスプレイの可能性** — 副ディスプレイでは座標がずれる
- `getGridArea()` は `ws.win.getBounds()` → グリッド領域計算 → これがAppleScript座標と不一致だとSnap位置がずれる
- **調査方法**: `system_profiler SPDisplaysDataType` + `screen.getAllDisplays()` + 同一ウィンドウの座標を3方式で比較

### 7.2 パッケージ版の権限
- ユーザーに「システム設定 → プライバシー → アクセシビリティ」で TerminalIN.app を許可してもらう
- Screen Recording 権限も同様
- 権限付与後はdaemon直接動作でフォールバック不要になる可能性

### 7.3 絶対にやってはいけないこと
- **リファクタリング禁止**: 変数名・関数名・IPCチャンネル名の変更は致命的バグの原因になった
- **最適化禁止**: hash-based skip、adaptive polling等が複数バグを生んだ
- **daemon.swift を書き換えない**: 初期リリース版が最も安定
- **複数ファイルの同時大規模変更禁止**: 1ファイルずつ、最小限の変更で、毎回テスト

## 8. 試行錯誤の記録

### うまくいったこと
- osascript `tell application "ターミナル" to activate` でターミナルを前面化
- `set index of window id N` でスナップしたウィンドウだけを最前面に
- daemon の `NSRunningApplication.activate()` + 80ms wait + `AXRaise` + `kAXMainAttribute`
- `acceptFirstMouse: true` + `alwaysOnTop: true`
- `browser-window-focus` での自動raise（再activateなしでループ防止）
- osascript `tell app to get id/name of every window` でタイトルフォールバック

### 失敗したこと
- `CGSOrderWindow` → macOS 26 で `err=1000`
- `AXRaise` 単独 → 他アプリのウィンドウを前面に持ってこれない
- **リファクタリング46cd073** → IPCチャンネル名不一致(`'grid-terminal-input'`→`'gi'`)、hash最適化でSnap壊れ、dbg削除漏れでクラッシュ
- `activate()` + 再activate → ループまたはz-order戻り
- パッケージ版daemon → AX/Screen Recording権限不足
