# 引き継ぎ: TiN — daemon TCC 問題の根本解決

> 作成日時: 2026-04-13 03:30
> ステータス: ブロック中 — daemon AX 権限がリビルドで壊れる

---

## 問題

daemon (Swift バイナリ) はリビルドするたびに CDHash が変わり、macOS TCC の Accessibility 権限が無効化される。ユーザーが手動で再追加する必要がある。`Contents/MacOS/` 配置でも TCC は daemon を個別バイナリとして管理する。

## 現状

- daemon が AX 権限あり → snap 8-56ms (高速)
- daemon が AX 権限なし → osascript fallback → 300-1500ms (遅い)
- daemon.swift を変更するたびにこの問題が再発

## 根本解決案

### A. Node.js N-API addon (推奨)
- AXUIElement 操作を C/ObjC で書いて N-API addon としてビルド
- Electron main process 内で実行 → TiN.app の TCC 権限を直接使用
- daemon バイナリ不要、TCC 問題消滅
- 実装: `native/ax-helper.mm` + `binding.gyp`

### B. 永続 osascript プロセス
- `osascript -l JavaScript` を daemon 代わりに起動、stdin/stdout で通信
- System Events 経由なので TCC 不要
- プロセス起動コスト (~200ms) をゼロに
- ただし System Events IPC は daemon AX より遅い

### C. daemon バイナリを固定
- daemon.swift を変更しない限りリビルドしない (現在の build-daemon.sh)
- 初回だけ手動追加、以降は安定
- daemon.swift に新機能を追加するたびに再追加が必要

## 現在のブランチ: main (82dcc01)

## daemon が使う機能
- list: CGWindowList (AX 不要)
- move: AXUIElement set position/size (AX 必要)
- raise: AXUIElement AXRaise (AX 必要)
- wobble: AX position set (AX 必要)
- verify: AX window existence check (AX 必要)

## 補足
- Spaces (仮想デスクトップ) 移動の CGS API は daemon から除去済み
- osascript の `name contains` 検索が 30+ ウィンドウで遅い (O(n))
- findWindow に CGWindowList position フォールバック追加済み
