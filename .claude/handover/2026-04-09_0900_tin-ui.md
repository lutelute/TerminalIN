## 1. プロジェクト概要

macOS用ターミナルワークスペースマネージャー。外部Terminal.appウィンドウをグリッド配置し、Workspaceサイドバーから管理する。

- **技術スタック**: Electron 41 + node-pty + xterm.js + Swift daemon (AX API)
- **リポジトリ**: https://github.com/lutelute/TerminalIN
- **インストール先**: `/Applications/TiN.app`

## 2. ファイル構成

```
main.js              — メインプロセス（~650行）
workspace.html       — サイドバーUI（~540行）
grid-overlay.html    — 半透明グリッド領域表示
grid-terminal.html   — 埋め込みxterm.jsターミナル
daemon.swift         — Swift AXウィンドウ操作デーモン
daemon               — コンパイル済みバイナリ
package.json         — name: "tin", productName: "TiN"
```

## 3. 今回のセッションで行ったこと

### 3.1 リネーム: TerminalIN → TiN
- package.json, workspace.html, appId全てTiNに変更

### 3.2 Grid Overlay実装
- `grid-overlay.html`: 半透明ウィンドウでグリッド領域を可視化
- `transparent: true` + `setIgnoreMouseEvents(true, {forward: true})` でクリックスルー
- 右端・下端・右下角にリサイズハンドル（`elementFromPoint`でハンドル検出→click-through解除→ドラッグ）
- リサイズはカスタムIPC (`resize-overlay`) でmain processがsetBounds

### 3.3 Raise/前面化の改善
- `alwaysOnTop` 完全廃止 → 通常ウィンドウ挙動
- Workspaceの`mousedown`で`raise-all`送信 → 全ターミナル+overlay+workspace前面化
- `raiseAllWorkspaceWindows`: ext→grid→overlay→workspace の順で前面化
- osascript fallbackの`activate`廃止（他アプリ全ウィンドウが前面に来る問題）→ System Eventsの`AXRaise`+`set frontmost`
- Raiseボタン(▣)削除（mousedown auto-raiseで不要）

### 3.4 Snap位置修正
- osascript fallback: `title.substring(0,20)` マッチ → `window id` マッチに変更（同名ターミナル問題）
- retile debounce: 16ms → 300ms（ディスプレイ間ドラッグ時の位置ドリフト防止）

### 3.5 マルチディスプレイ位置問題の根本修正 (最重要)
- **原因**: `transparent: true`ウィンドウの`getBounds()`がmacOSマルチディスプレイで不正確な値を返す（1440px yオフセット）
- **対策**: `getGridArea()`を`overlay.getBounds()`依存から`workspace.getBounds() + gridWidth`計算に変更
- `gridWidth`をworkspace stateに保存、overlayリサイズ時に更新
- overlay位置の同期も全てworkspace boundsから計算

### 3.6 UI リデザイン
- タイトルバー: 52px高さ、クリックで名前編集
- セクション: GRID(薄紫bg)、AVAILABLE(薄青bg)、CONSOLE(下部固定・折り畳み)
- フラットデザイン: モノスペースフォント、ボーダーレスボタン
- サイドバー⇔ターミナル gap: 4px→12px

## 4. 既知の問題・注意点

### ビルド・インストール
- `cp -R` で `.app` を上書きすると asar が古いまま残る → **必ず `rm -rf` してから `cp -R`**
- ビルド後にasar抽出してdiffで検証すること（issue #1参照）

### transparent ウィンドウ
- `overlay.getBounds()` は信用しない。常に workspace bounds + gridWidth で計算
- overlay作成は `win.once('show', () => setTimeout(createGridOverlay, 100))` で遅延必要

### マルチディスプレイ
- sleep/wake後にウィンドウ位置がリセットされることがある
- スナップ済みターミナルのwindowNumberが変わる可能性

- ユーザーは「claudeっぽいデザイン」を好まない（丸すぎる、色が強すぎる）
- Sidebar Terminalsはほとんど使わない → CONSOLE としてリネーム、下部固定・折り畳みデフォルト
- Electron devモードでは問題なくてもパッケージ版で動かないケースが頻発 → 必ずパッケージ版で動作確認
- リモートデバッグ: `--remote-debugging-port=9222` で起動すればMCP接続可能
- `app.commandLine.appendSwitch('remote-debugging-port', '9222')` はパッケージ版で効かない → 起動引数で渡す
